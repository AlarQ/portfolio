## Verdict Rationale

`reopen` — two FRs are `partial`:
1. **FR-3**: the "low-MVP-Progress → muted tone" rule stated in the FR text and Data Model constraints has no implementation; `projectPresentation` is a pure function of `status` only. This is a genuine spec-vs-code gap, not a documentation nuance — the current single authored Project (`mvpProgress: 80`, `status: "in-progress"`) never exercises the missing branch, so nothing today visibly breaks, but a future low-progress, non-`exploring` Project would silently render with a non-muted tone contrary to spec.
2. **FR-5**: `zoom-200-tablist` and `reduced-motion-swap` are unverified by any test — implementation exists (motion-safe classes, overflow/scroll-snap rail) but is unproven, unlike the FR's other scenarios which are unit/e2e-covered.

## Punch List to Close

1. **FR-3 (blocker)** — Extend `projectPresentation` to accept `mvpProgress` (or add a small pure helper `isLowMvpProgress(mvpProgress): boolean` with an agreed threshold) and fold it into the tone decision so a non-`exploring` Project with low `mvpProgress` also resolves to `muted`. Add a unit test asserting this (e.g., `status: "in-progress", mvpProgress: 5` → `tone: "muted"`) and a Storybook state exercising it. Est: 1-2 hrs.
2. **FR-5 (should-fix before complete)** — Add an e2e assertion for `reduced-motion-swap` (Playwright supports `page.emulateMedia({ reducedMotion: 'reduce' })`) confirming the swap wrapper carries no active transition class/duration when reduced motion is requested, and a `zoom-200-tablist` check (e.g., set a `deviceScaleFactor`/CSS zoom via `page.evaluate` or viewport-shrink proxy) confirming the tablist stays one row with all tabs reachable via keyboard even when visually overflowing. Est: 2-3 hrs.
3. **Coverage note (non-blocking)** — Author a second `Project` in `projects.ts` (or an explicit fixture-driven unit/e2e test independent of live data) so `pill-switch-summary`'s multi-pill path is exercised by the real dataset instead of being e2e-skipped via `test.skip(projects.length < 2, …)`.

Once (1) and (2) are closed and verified green, this spec is complete.
